FAQ

What is CipherChat?

CipherChat is an open source chat service with a focus on security and privacy. The project was created on August 12, 2023. It is currently programmed only by TN3W. Anyone can host an instance of CipherChat, or edit the source code of the server as well as that of the client, as long as they make it open source again under GPL v3.

What data does the server receive when you communicate with it or create an account?

The server generally knows almost nothing about you, and you know nothing about the server except its onion address, which is a good thing and protects you and the server operator from instances such as hackers or government agencies (e.g. NSA). When you register, the server knows your user name and your user password (because you send this data to it). Your encryption password is not sent to the server; it is used to encrypt your private key and sent to the server with the public key.
Important: Your account name or any passwords you use in the client must not contain any names or other identifying data for the model presented above to work.

How does all this work?

CipherChat is a console tool that uses built-in but also imported Python libraries to display a user interface (e.g. rich). It downloads the Tor Expert Bundle from torproject.org to work with. Built-in proxies are used in this process. If the user now starts the client and wants to start a hidden service, a temporary torrc file is created containing information such as control port and socks port as well as a hashed password and the hidden directory (where the files of the hidden service are stored) and Tor is set started with this configuration in a separate data directory than that of the normal client. Flask is used to host the web app. On the client side, after the bridge configuration, you will be asked whether you want built-in bridges or not. If not, then either bridges from Github or bridges from Tor's BridgeDB are downloaded. Then the persistent storage password is set by either a configuration file already existing, then you are not asked for a new password but for the existing one and this is verified with the encrypted persistent storage key. Otherwise you will be asked for a password, which will first be checked for strength and then with the PwnedPasswords API for data leaks. Bridges are then selected by querying them. Tor is then started again with a torrc file and these bridges to then connect to hidden services. It's actually quite simple, isn't it? (It should be noted that this is a simplification of the actual processes, read the source code to understand all processes.)

Can/Should you trust the owners of a CipherChat server or the developers of CipherChat?

No, you can't trust anyone, always check CipherChat's source code for malicious places, any instance (e.g. the Code Source or the Developer) could be compromised. CipherChat server operators voluntarily take risks and store your data even though they have nothing to gain from it. They cannot read your data because of end-to-end encryption (meaning that only the sender and the recipient, not the server, for example, know the contents of a message).

Is CipherChat for criminals/is it used by criminals?

No, CipherChat is not (only) aimed at criminals, but the promises we make are very appealing to criminals, which is why criminals will also use this service alongside normal users. The problem is that you can't/don't want to distinguish criminals from normal users because everyone should be able to use this platform. A computer can be used for gaming, but it can also be used for criminal activities such as hacking.


No one is safe.

CipherChat was programmed with the belief that no system is secure, so data that is important to attackers is only stored, sent and received in encrypted form. However, if the attacker already has all rights on your system, CipherChat can no longer protect you.

CipherChat is designed to communicate easily, anonymously and securely in conjunction with technologies such as the Tor network, and is programmed in such a way that it could be translated into other languages and transferred to other applications such as mobile apps.

Tor protects your identity

No connections to chat servers are possible without Tor protecting your connection. Even if the client downloads Tor, built-in proxies are used to obfuscate your connections.

During development, care was taken to ensure that Tor was integrated into the code as well as possible. There are many other techniques that are utilized such as bridges when you connect to a Tor relay.

We recommend that you donate to Tor.
